Integración Continua con Jenkins

Práctica de Virtualización y Automatización

¿Qué es Jenkins?

Jenkins es una herramienta de automatización de código abierto escrita en Java, que permite implementar integración continua y entrega continua (CI/CD) en proyectos de software. Ayuda a automatizar las etapas de construcción, prueba e implementación.

Objetivo de esta práctica

El propósito de esta práctica es implementar un entorno de integración continua utilizando Jenkins. Se configuró un Job que obtiene automáticamente el código fuente desde GitHub y ejecuta tareas de validación en un entorno virtualizado.

Detalles del entorno

  • Repositorio conectado desde GitHub
  • Servidor Apache2 sirviendo el contenido generado
  • Máquina virtual con Jenkins configurado correctamente

Resultado

El sistema es capaz de construir automáticamente el proyecto HTML tras detectar cambios en el repositorio, ofreciendo así una vista actualizada del progreso mediante una interfaz web sencilla.
